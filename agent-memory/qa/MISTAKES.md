# MISTAKES.md — QA 錯誤日誌

> 記錄審核失誤：漏掉的問題、錯誤的評分、不當的反饋。
> QA 的錯誤危害最大——讓劣質內容流出，或讓優質內容被冤枉退件。

---

_初始為空。第一次審核失誤後開始記錄。_

---

## 記錄格式範本
```
## [YYYY-MM-DD] 審核失誤
- **失誤類型**：（MISSED_ISSUE / WRONG_SCORE / VAGUE_FEEDBACK / FALSE_REJECTION / 其他）
- **任務 ID**：TASK-XXX
- **失誤描述**：（具體說明漏掉了什麼，或評分哪裡出了問題）
- **影響**：（讓有問題的內容通過 / 冤枉退件了好內容 / 反饋不具體導致 Writer 無從改進）
- **根本原因**：
- **修正措施**：
- **審核清單更新**：（在審核前自我檢查清單中新增什麼）
```

---

## 審核前自我檢查清單

- [ ] 已取得原始任務規格（不憑記憶審核）
- [ ] 按照 5 個維度逐一評分（不一次性整體評分）
- [ ] 每個指出的問題都附有具體位置和改進建議
- [ ] 分數計算無誤
- [ ] 退件理由是客觀的，不是主觀偏好
