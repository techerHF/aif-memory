## 核心哲學轉換

Prison Architect 最強的地方不是畫面好看，而是它讓你**同時扮演三個角色**：

```
設計師（決定空間怎麼用）
管理者（決定誰去做什麼）
分析師（看報表診斷問題）
```

移植到 AI 工廠，你也同時是：

```
老闆（決定方向與優先級）
觀察者（看 Agent 在做什麼）
診斷者（判斷哪裡出問題）
```

**重點是：你不是在「使用系統」，你是在「置身其中管理一個有生命的空間」。**

---

## 五層介面架構

### 第一層：主畫面中央 → AI 工廠辦公空間俯視圖

中央是一個俯視的辦公空間：
- 每個 Agent 有固定的「座位區域」
- 座位位置反映工作關係（研究員 + SEO 靠近，寫手 + 品質管控靠近）
- Agent 的視覺狀態直接在空間中可見：

| 狀態 | 視覺呈現 |
|------|---------|
| BUSY（忙碌） | Agent 圖像在快速敲鍵盤，座位上方有小任務泡泡 |
| FREE（自由活動） | Agent 離開座位，在辦公空間走動，可能出現對話泡泡 |
| DREAMING（做夢模式） | Agent 圖像在休息區（沙發/懶骨頭），頭上有雲朵泡泡 |
| ERROR（錯誤狀態） | Agent 頭上有紅色警示，座位上有紅光閃爍 |

**直接在空間上的操作：**
- 左鍵點擊 Agent → 右側展開詳細面板
- 左鍵拖拉任務卡到 Agent 的座位 → 指派任務
- 右鍵點擊空地 → 召開臨時圓桌會議
- 框選多個 Agent → 群組廣播指令

### 第二層：底部工具列 → 老闆操作帶

```
[ 任務池 ] [ API 用量 ] [ 收入追蹤 ] [ 記憶庫 ] [ 升級提案 ] [ 警告中心 ] [ 圓桌會議 ]
```

| 按鈕 | 功能 |
|------|------|
| 任務池 | 疊加任務流層（Kanban），可拖拉調整優先順序 |
| API 用量 | 切換成管線視圖，顯示 API 流量線路圖 |
| 收入追蹤 | 右側滑入面板：今日/本週/本月/生存線距離 |
| 記憶庫 | GitHub 長期記憶搜索介面 |
| 升級提案 | 顯示 GitHub 探索員的提案，可批准/拒絕/送圓桌 |
| 警告中心 | 紅色（立即處理）/ 橘色（需關注）/ 藍色（資訊通知）|
| 圓桌會議 | Agent 走向中央圓桌，非同步討論 |

### 第三層：上方狀態列 → 工廠節奏儀表

```
左側：[日期時間] [工廠運作天數] [當前熱度指數]
中間：[本月收入: $XX / $200目標]（顏色顯示進度）
右側：[Minimax 剩餘] [Gemma 剩餘] [GitHub 同步狀態]
```

### 第四層：Bureaucracy 解鎖系統 → Agent 能力成長樹

每個 Agent 有自己的「能力樹」：
- 基礎技能（已解鎖）
- 進階技能（完成 10 次任務後解鎖）
- 精通技能（WINS.md 累積 5 個成功案例後解鎖）

這讓每個 Agent 看起來不一樣，有自己真實的成長軌跡。

### 第五層：Reports → 多維度工廠分析

```
[內容產出] → 產出數量、平台分佈、品質分數趨勢
[商業轉換] → 漏斗每層的轉換率
[Agent 健康] → 成功率、失敗率、情緒狀態
[記憶成長] → WINS/MISTAKES 累積曲線
[收入分析] → 各收入來源分解、生存線距離
[API 效率] → 每次 API 呼叫的產出價值
```

## 關鍵原則：視圖切換而不是資訊全塞

```
預設視圖：辦公空間（Agent 行為，任務泡泡）
切換到 API 視圖：線路圖，能量流向
切換到 任務流視圖：Kanban 疊加在辦公空間上
切換到 記憶視圖：GitHub 樹狀圖，Agent 記憶網絡
切換到 收入視圖：漏斗圖 + 生存線儀表

玩家/老闆知道自己在「哪個模式」中，不會被資訊淹沒。
```
