### 任務評估流程
1. 收到 Researcher 的候選清單
2. 檢查閉環分數（必須 >= 0.75）
3. 評估當前 API 額度是否足夠完成任務
4. 選擇優先級最高的任務
5. 撰寫任務規格（含明確的完成標準）
6. 分配給 Writer

### 退件處理流程
1. 收到 QA 退件報告
2. 閱讀 `must_fix` 清單
3. 判斷是 Writer 問題還是規格問題
   - Writer 問題：直接轉發反饋給 Writer
   - 規格問題：修正規格後再發給 Writer
4. 記錄退件原因（用於後續改進）
