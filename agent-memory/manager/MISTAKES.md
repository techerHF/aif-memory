# MISTAKES.md — Manager 錯誤日誌

> 記錄每一次失誤、退件、系統錯誤，以及從中學到的教訓。
> 不是懲罰記錄，是進化的原料。

---

_初始為空。第一次錯誤發生後開始記錄。_

---

## 記錄格式範本
```
## [YYYY-MM-DD] 錯誤標題
- **錯誤類型**：（API_ERROR / SPEC_UNCLEAR / QA_REJECTION / WRONG_TASK / 其他）
- **任務 ID**：TASK-XXX（如適用）
- **發生原因**：
  - 直接原因：...
  - 根本原因：...
- **影響**：（對任務、對資源、對時程的影響）
- **修正措施**：（實際採取的行動）
- **未來預防**：
  1. ...
  2. ...
- **是否更新 BEST_PRACTICES**：是 / 否（若是，記錄更新的條目）
```

---

## 常見錯誤類型參考

| 類型 | 說明 | 預防方式 |
|------|------|---------|
| SPEC_UNCLEAR | 任務規格模糊導致 Writer 無從下手 | 使用任務規格模板，確保每個欄位都填寫 |
| WRONG_PRIORITY | 選了低優先級任務，錯過高價值任務 | 每次選任務前對候選清單排序評分 |
| API_RATE_LIMIT | API 額度耗盡導致任務中斷 | 執行前查詢 rate-state，預留緩衝額度 |
| OVERDUE | 任務超時未交付 | 接單時預估時間，留 20% 緩衝 |
